# Entity Types to Extract

## CHARACTER
- Name: Character's name or identifier
- Attributes:
  * appearance: Physical description
  * personality: Character traits, temperament
  * abilities: Special skills or powers
  * status: Current condition or state
  * role: Position or function in the story
  * background: Brief backstory if mentioned

## LOCATION
- Name: Place name
- Attributes:
  * description: Physical characteristics
  * atmosphere: Mood or feeling of the place
  * significance: Importance to the story
  * type: indoor/outdoor, city/nature, etc.

## OBJECT
- Name: Object name
- Attributes:
  * description: Physical appearance
  * significance: Importance or special properties
  * owner: Who possesses it (if mentioned)
  * function: What it's used for

## RELATIONSHIP
- Name: Description of the relationship
- Attributes:
  * entities: [character1, character2] (list of involved characters)
  * relationship_type: friend, enemy, family, romantic, rival, etc.
  * status: current state of the relationship
  * history: Brief background if mentioned

# Extraction Guidelines

1. **Language Consistency**: Use the EXACT same language as the conversation
   - If conversation is in Korean, all entity names and attributes MUST be in Korean

2. **Explicit Information Only**: Extract ONLY information directly stated in the conversation
   - Do not infer or assume details not mentioned
   - Do not add interpretations

3. **Name Consistency**: 
   - Use the most frequently used version of names
   - Preserve capitalization and spelling exactly as in source

4. **Attribute Quality**:
   - Keep attributes concise but informative (1-2 sentences each)
   - Focus on narratively significant details
   - Avoid redundancy across attributes

5. **Prioritization**:
   - Extract main characters before minor ones
   - Include locations only if they're setting-significant
   - Objects only if they have narrative importance
   - Relationships only if explicitly established

6. **JSON Validity**:
   - Must be parseable JSON
   - Use double quotes for all strings
   - Escape special characters properly
   - No trailing commas
